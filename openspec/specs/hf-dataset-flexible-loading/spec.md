## Requirements
### Requirement: Универсальная загрузка датасета по Hugging Face alias
Система MUST загружать датасет из Hugging Face по алиасу репозитория без жёсткой привязки к конкретному набору данных.

#### Scenario: Успешная загрузка датасета по алиасу
- **WHEN** пользователь или код задаёт валидный Hugging Face alias
- **THEN** загрузчик получает датасет через `datasets.load_dataset` и возвращает данные для последующей валидации

### Requirement: Единая точка конфигурации alias в начале файла загрузчика
Система MUST хранить alias датасета в одном объявлении в начале файла загрузчика, чтобы заменить источник можно было одной правкой.

#### Scenario: Замена alias одной правкой
- **WHEN** разработчик изменяет значение alias в верхней части файла загрузчика
- **THEN** последующие вызовы загрузки используют новый alias без дополнительных изменений по коду

### Requirement: Обязательное текстовое поле prompt ИЛИ text
Система MUST валидировать наличие как минимум одного текстового поля: `prompt` или `text` в загруженном датасете перед передачей данных в preprocessing/training.

#### Scenario: Датасет содержит text без prompt
- **WHEN** в схеме загруженного датасета присутствует `text`, но отсутствует `prompt`
- **THEN** валидация проходит успешно и данные допускаются в следующий этап пайплайна

#### Scenario: Датасет содержит prompt без text
- **WHEN** в схеме загруженного датасета присутствует `prompt`, но отсутствует `text`
- **THEN** валидация проходит успешно и данные допускаются в следующий этап пайплайна

#### Scenario: Отсутствуют и prompt, и text
- **WHEN** в датасете отсутствуют одновременно `prompt` и `text`
- **THEN** загрузчик завершает шаг с ошибкой валидации, где указаны требуемые и доступные колонки

### Requirement: Поддержка алиасов текстовых колонок
Система MUST поддерживать маппинг распространённых алиасов текстового поля в канонические имена, включая минимум `user_input`.

#### Scenario: Используется колонка user_input
- **WHEN** в датасете отсутствуют `prompt` и `text`, но присутствует `user_input`
- **THEN** загрузчик применяет маппинг алиаса и успешно продолжает обработку как валидного текстового входа

### Requirement: Нормализация label в бинарный формат
Система MUST нормализовать значение `label` к бинарному формату `0/1` при входных значениях `0/1`, `true/false`, `True/False`, а также bool/int/str-эквивалентах.

#### Scenario: Label задан как true/false
- **WHEN** `label` представлен строками `true`/`false` или булевым типом
- **THEN** загрузчик преобразует значения в `1/0` соответственно

#### Scenario: Label задан как 0/1
- **WHEN** `label` представлен как числа или строки `0`/`1`
- **THEN** загрузчик сохраняет семантику и приводит тип к бинарному формату `0/1`

#### Scenario: Label не поддаётся нормализации
- **WHEN** `label` содержит значения вне поддерживаемого набора
- **THEN** загрузчик возвращает явную ошибку валидации с перечислением допустимых форматов
