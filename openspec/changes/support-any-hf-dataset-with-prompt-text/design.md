## Context

Текущий датасетный слой проекта классификатора атак на LLM должен поддерживать быстрый переход между источниками данных для экспериментов и обучения. На практике используется Hugging Face, но реализация загрузчика ограничена частным сценарием и не гарантирует наличие обеих обязательных колонок (`prompt`, `text`). Также alias датасета часто «размазан» по коду, что повышает стоимость замены датасета и риск рассинхронизации.

Ограничения:
- Не ломать существующий контракт downstream-компонентов без необходимости.
- Сохранить совместимость с текущим стеком `datasets`.
- Обеспечить понятные ошибки валидации схемы на этапе загрузки, а не в середине пайплайна.

## Goals / Non-Goals

**Goals:**
- Сделать загрузчик универсальным для любого Hugging Face dataset alias.
- Явно закрепить обязательные колонки `prompt` и `text` в схеме.
- Вынести alias в одно место в начале файла загрузчика для быстрой замены.
- Стандартизировать поведение ошибок при отсутствии обязательных колонок.

**Non-Goals:**
- Редизайн полного training pipeline.
- Введение новой системы конфигурации на уровне всего проекта.
- Изменение бизнес-логики инференса и rule-based модуля.

## Decisions

1. Единый alias-константный параметр в начале загрузчика
- Решение: объявить значение HF alias один раз в верхней части `loader.py` (например, `HF_DATASET_ALIAS = "neuralchemy/Prompt-injection-dataset"`) и использовать его по умолчанию в функции загрузки.
- Почему: минимальная точка изменения, прозрачность для разработчика, отсутствие необходимости искать строковый литерал по проекту.
- Альтернатива: передавать alias только через CLI/ENV.
  - Причина отказа: в текущем объёме это избыточно; требование пользователя — быстро заменяемый alias в начале файла.

2. Универсальная загрузка через `datasets.load_dataset(alias, ...)`
- Решение: не хардкодить конкретные split/наборы колонок для одного источника; поддерживать любой alias с валидацией после загрузки.
- Почему: соответствует требованию «абсолютно любой датасет с Hugging Face».
- Альтернатива: allowlist поддерживаемых датасетов.
  - Причина отказа: противоречит требованию универсальности.

3. Жёсткая проверка обязательных колонок `prompt` и `text`
- Решение: обновить схему/валидатор, чтобы обе колонки были обязательны; при отсутствии — детальная ошибка с перечислением найденных колонок и отсутствующих полей.
- Почему: раннее обнаружение несовместимых датасетов, предсказуемый контракт для preprocessing/training.
- Альтернатива: автоматически маппить «похожие» колонки (`instruction`, `input`, `question`) в `prompt`.
  - Причина отказа: повышает риск скрытых ошибок и неявной трансформации данных.

4. Минимальные правки downstream
- Решение: сохранить формат выходного DataFrame/датасета, добавив только гарантии присутствия `prompt` и `text`.
- Почему: снижает риск регрессий в обучении и метриках.

## Risks / Trade-offs

- [Риск] Некоторые HF-датасеты не содержат `prompt` или `text` в явном виде → [Смягчение] явная ошибка валидации с инструкцией о необходимости привести схему.
- [Риск] Универсальная загрузка может требовать настройки split (`train`, `test`) для конкретного датасета → [Смягчение] поддержать параметр split с дефолтом и понятной ошибкой при недоступном split.
- [Trade-off] Жёсткие обязательные поля снижают «магическую» совместимость, но повышают воспроизводимость и прозрачность.
- [Trade-off] Alias в коде проще менять вручную, но менее гибок, чем централизованный конфиг.

## Migration Plan

1. Обновить `loader.py`: добавить единый alias в начале файла и универсальную загрузку.
2. Обновить `schema.py`/`validator.py`: сделать `prompt` и `text` обязательными.
3. Обновить тесты датасетного слоя под новые требования.
4. Прогнать smoke/eval сценарии, убедиться в отсутствии регрессий.
5. Rollback: вернуть предыдущую версию загрузчика и схемы из VCS, если критические пайплайны перестанут проходить.

## Open Questions

- Нужно ли в ближайшем этапе вынести alias в отдельный конфиг-файл проекта, сохранив в `loader.py` только fallback?
- Нужна ли дополнительная поддержка автоматического маппинга колонок как опциональная функция (не по умолчанию)?
